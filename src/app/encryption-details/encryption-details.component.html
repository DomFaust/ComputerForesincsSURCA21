Welcome to the world of Encryption!

Below, you will find some of the basic terms that are used when discussing this field.

Plaintext
Ordinary, intelligible text, that is not formatted or written in code Cipher
An algorithm to encode and disguise plaintext, or to reveal encoded ciphertext Ciphertext
Text that has been encrypted with the use of a cipher Encryption
The process of securing digital data by using mathematical algorithms to encode the information Keys
In encryption, it is a chunk of information, usually a random string of numbers or letters, that can be used to encode or decode data Hash
A value derived from a hashing encryption process that cannot be used to reverse engineer the original plaintext

Welcome to the PNG section!

This section focuses on identifying and extracting information from these files.

PNG
Stands for Portable Network Graphics. It is a graphics file format that supports image files. PNG File Structure
A PNG file consists of a signature, followed by a series of chunks. All of which is defined using bytes. PNG Signature
The first 8 bytes of every PNG file are identical, and is referred to as PNG's signature. It can be used to signify that the file being examined is a PNG file. This series of bytes are 89 50 4E 47 0D 0A 1A 0A, or 137 80 78 71 13 10 26 10 in decimal. PNG Chunk
These bytes detail attributes about the PNG file. Each chunk is arranged with the following components: Length, Chunk Type, Chunk Data, CRC. Length
A 4-byte unsigned integer giving the number of bytes in the chunk's data field. The length does not count itself, the chunk type code, or the CRC. Chunk Type
A 4-byte code consisting of ASCII characters, that can be used to obtain a better description and examination of the PNG file. Chunk Data
The chunk data is a series of data bytes that describe the chunk type. CRC
Stands for the Cyclic Redundancy Check. It is a 4-byte value that is calculated using the preceding bytes in the chunk, including the chunk type code and the chunk data fields. The length field is not used to calculate the CRC. IHDR Chunk
It is the first chunk following the file signature. This chunk contains information regarding the image's height, width, and bit depth. IEND Chunk
It is the last of the series of chunks that follow the file signature. This chunk is just used to symbolize the end of the image file.

MD5 (Message-Direct Algorithm)
A hashing algorithm that accepts a message as an input and returns a value as its output. That value can authenticate the original message. Purposes of the MD5
MD5 has been used to verify that a file has been unaltered. It has also ben used to securely store passwords.
